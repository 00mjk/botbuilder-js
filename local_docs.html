Welcome to the second hackathon for the v4 version of the BotBuilder SDK. To get started building bots using Node.js SDK v4, follow the setup instructions outlined below.

Connect to our internal feed

The NPM modules for the SDK are located on a private internal feed that all Microsoft employees should have access to. To access this feed you will need to first install and run an auth helper which will save your credentials for accessing the feed. From the current folder type:

Windows
  1. npm install -g vsts-npm-auth --registry https://registry.npmjs.com --always-auth false
  2. vsts-npm-auth -config .npmrc

macOS
  1. Go to: https://fuselabs.visualstudio.com/Intercom/_packaging?feed=FuseNPM&_a=feed
  2. Click the blue "Connect to feed" button in the upper right
  3. Click "npm" option in left nav
  4. At the bottom of the pop-up window copy the text under the section: "OSX/Linux users: Add credentials to your user .npmrc manually"
  5. Open terminal and open the file vi ~/.npmrc and paste in the auth code from the previous step & save
  6. Navigate back to the Node.js SDK directory cd ~/code/BotBuilder-Dev/js and run npm install

That should install the auth helper and then configure your credentials for our private feed. If for some reason it fails, first ensure that you're in the directory containing the .npmrc file and then consult this troubleshooting guide.

Install NPM modules

Once you have your auth credentials setup you can install the NPM modules needed to run the samples and build your own bots. Simply type npm install from the current folder.

Install Emulator

To test and debug your bot you'll want to install the latest version of the BotFramework Emulator. You can find instructions for debugging your bot using the emulator here.

Run Examples

Get started by running one of the various examples included here.
